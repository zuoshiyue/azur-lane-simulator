# 双阵容系统实施计划

## 需求分析

### 当前状态
- 单一阵容：6 个槽位（先锋 3 + 主力 3）
- 潜艇编队单独计算
- 无法同时查看多个阵容

### 目标状态
- **双海上阵容**：阵容 A + 阵容 B（各 6 槽位）
- **潜艇阵容**：1-2 个（各 6 槽位）
- 总计：2 个海上 + 1-2 个潜艇 = 最多 4 个阵容

## 实施步骤

### 步骤 1: 数据结构修改
```typescript
// 当前
interface Fleet {
  id: string;
  name: string;
  characters: (Character | null)[];
  type: FleetType;
}

// 修改为
interface FleetGroup {
  id: string;
  name: string;
  fleets: {
    surface1: Fleet;  // 海上阵容 1
    surface2: Fleet;  // 海上阵容 2
    sub1: Fleet;      // 潜艇阵容 1
    sub2?: Fleet;     // 潜艇阵容 2（可选）
  };
}
```

### 步骤 2: 修改 FleetSimulator 组件
- 支持多阵容显示
- Tab 切换或并排显示
- 分别计算每个阵容的战力

### 步骤 3: 修改推荐系统
- 推荐输出包含 2 个海上阵容
- 潜艇阵容单独推荐
- 考虑角色不重复

### 步骤 4: 修改阵容保存/导出
- 保存完整的 FleetGroup
- 导出格式包含所有阵容

## 简化方案（推荐）

考虑到改动幅度，建议先实施简化版本：

### Phase 1 - 阵容标签页
- 添加"阵容 1" / "阵容 2"切换
- 每个阵容独立保存
- 快速复制阵容到另一个

### Phase 2 - 并排显示
- 移动端：标签页切换
- 桌面端：并排显示两个阵容

### Phase 3 - 推荐系统适配
- 推荐系统输出两套阵容
- 一键应用到对应槽位

## 验收标准

- [ ] 支持创建/切换两个海上阵容
- [ ] 每个阵容独立保存和加载
- [ ] 推荐系统提供两套阵容建议
- [ ] 阵容导出包含所有阵容数据
- [ ] 移动端友好显示

## 预计时间
- Phase 1: 2-3 小时
- Phase 2: 2-3 小时
- Phase 3: 3-4 小时
